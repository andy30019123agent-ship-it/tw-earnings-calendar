# 報告「決策工具化」資料契約（2026-07-09）

> 目的：把研究報告從「研究文件」升級為「決策工具」。此檔是 **SOP（內容）、前端渲染、報告作者** 三邊的共同契約。前端讀這些結構化欄位渲染決策 UI；舊報告沒有這些欄位時一律**優雅降級（該區塊不顯示、不破版）**，正文照常。

## 1. 新增 frontmatter（新報告必填；舊報告無→前端降級）

```yaml
---
id: 2454
name: 聯發科
date: 2026-07-09
type: 快報            # 快報(事前) | 法說會/財報(事後) | 產業分析
event_date: 2026-07-30
# ↓↓↓ 新增「決策」區塊（決策儀表板＋底部常駐條讀它）
decision:
  stance: 觀望        # 偏多 | 偏空 | 觀望 | 逢高減碼 | 避開（今日結論色塊）
  price: 1180         # 撰寫時現價（元）
  fair_low: 950       # 合理價-保守
  fair_mid: 1150      # 合理價-中性
  fair_high: 1400     # 合理價-樂觀
  entry: "950–1080"   # 建議進場區（字串，可含區間或條件）
  stop: 880           # 停損（元或字串）
  take_profit: "1400+"# 停利
  timeframe: 中線財報驗證   # 短線題材 | 中線財報驗證 | 長線轉型
  confidence: 3       # 1–5（誠實反映不確定性，擇時/股價類別別給高分）
  invalidation: "Q2 毛利率跌破 45% 或旗艦晶片丟單"   # 翻盤條件（一句）
  next_event: "2026-07-30 法說會"                    # 下一關鍵事件＋日期
# ↓↓↓ 新增「關鍵數字卡」（橫滑小卡，每張帶判斷標籤）
key_stats:
  - { label: "本益比", value: "18.5x", status: 合理 }   # status: 便宜|合理|偏貴|危險
  - { label: "股價位置", value: "半年 +32%", status: 偏貴 }
  - { label: "營收 YoY", value: "+11%", status: 合理 }
  - { label: "毛利率", value: "48.2%", status: 合理 }
  # 3–6 張，挑最能支撐/挑戰結論的數字；把「股價 vs 基本面」的矛盾放進來
---
```

- 所有數字仍遵守 SOP「資料蒐集鐵律」：附來源/日期、查無標查無、不確定標 🟡。
- `decision`/`key_stats` 任何欄位缺失，前端該格顯示「—」或整塊隱藏，不得破版。

## 2. 內文章節新順序（決策價值高的前置）

1. `## 0. 操作結論`（新增，置頂）＝把 decision 用白話寫成 3–5 行：立場一句＋為什麼＋進場/停損/停利/時間框架/翻盤條件。**這是報告入口。**
2. `## 一句話總結`（壓 3 行內）
3. `## 關鍵矛盾`（新增，可選）＝這檔最該先看清的一件事（如股價 vs 基本面脫鉤）
4. `## 估值面`：**固定給三情境合理價（保守/中性/樂觀）**＋各自假設，不只講 PE
5. `## 催化劑時間軸`：每個事件加「**預期值 vs 驚喜門檻**」（例：Q2 毛利 45–47% 符合、48%+ 驚喜）
6. `## 多空評估`：左右對照（多方論點 / 空方論點）＋**「我比較信哪邊＋權重」**
7. `## 財務品質紅旗`
8. `## 同業比較表`：除價格位置，加「題材純度／估值貴度／基本面兌現度」
9. `## 進階洞察`（原「深度延伸」改名：供應鏈地圖／想像情境／非共識；**每個非共識點加「若錯了會看到什麼」**）
10. `## 後勢預測與觀察點`（附信心分）＋`## 來源彙整`
- 結尾加 `## 下次更新只看三件事`（3 個 bullet，讓回頭追蹤不用重讀全文）。

## 3. 事前 vs 事後 重點差異

- **事前快報**：核心＝「預期差清單」——市場期待什麼／財報·法說要驗證什麼／失望點是什麼（幫 Andy 準備事件交易框架）。
- **事後報告**：核心＝「預測回驗＋立場是否改變」——不是重寫公司介紹，是判斷原 thesis 升級或破功。

## 4. 前端渲染要點（呈現方式 1–4）

- **決策儀表板**（報告頁頂、重點卡之上）：讀 `decision`，一屏顯示 立場色塊／現價／合理區間／進場-停損-停利／時間框架／信心（用 5 格條或儀表，非純文字）／翻盤條件。缺 decision→整塊不顯示。
- **關鍵數字卡**：讀 `key_stats`，橫向可捲小卡，每張含 label／value／status 色標（便宜=偏多色、合理=中性、偏貴/危險=警示；**不得用 --up/--down 紅綠**，用 score/中性/accent 系）。
- **章節分「先看／深入」兩組**：ReportToc pill 分兩排。先看＝操作結論/一句話/關鍵矛盾/估值/催化劑/多空；深入＝財務紅旗/同業/進階洞察/預測/來源。
- **手機底部常駐條**：sticky bottom bar 顯示 立場＋停損＋下一事件（讀 decision），滑到深處仍見主判斷；缺 decision→不顯示。
- **進階洞察預設折疊**（`<details>` 或收合），其餘照常。
- **信心** 5 格條、**多空** 左右對照卡。emoji 標記（✅🔮🟡）弱化成小標籤不搶重點。
- ⚠️ 全部走 MASTER token、禁裸 hex、紅漲綠跌只給漲跌、觸控 44px、間距 4/8、字級 ≥14px、尊重 reduced-motion。舊報告（無新 frontmatter）一律降級不破版。
